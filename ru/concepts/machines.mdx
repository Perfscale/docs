## Обзор

**Машина** — это любой хост (bare metal сервер, виртуальная машина или контейнер), на котором запущен агент perfscale и выполняются тестовые скрипты. Машины принадлежат **тенанту** (workspace) и ограничены его областью видимости.

## Статусы

| Статус | Значение |
|--------|----------|
| `online` | Агент подключён, готов принимать задачи |
| `offline` | Агент недоступен |
| `busy` | В данный момент выполняет задачу |
| `maintenance` | Вручную выведен из ротации |

## Поля

| Поле | Тип | Описание |
|------|-----|----------|
| `name` | string | Человекочитаемое отображаемое имя |
| `hostname` | string | DNS-имя или IP, используемые агентом |
| `ipAddress` | string? | Публичный IP (опционально) |
| `os` | enum | `linux` \| `windows` \| `macos` |
| `arch` | enum | `amd64` \| `arm64` |
| `cpuCores` | int? | Количество ядер CPU |
| `memoryGb` | int? | Объём оперативной памяти в гигабайтах |

## Регистрация машины

Машины регистрируются через API или административную консоль. Каждая машина привязана к тенанту и обменивается данными через постоянное WebSocket-соединение (появится в будущих версиях — пока управляется вручную).

## Выбор машины

При запуске задачи perfscale выбирает онлайн-машины в workspace тенанта. Если доступно несколько машин, нагрузка распределяется равномерно.
